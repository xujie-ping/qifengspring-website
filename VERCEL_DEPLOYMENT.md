# Vercel Deployment Guide

This project is ready for Vercel deployment as a Next.js application.

## 1. Important Production Note

The current admin system stores content in local files:

- `data/cms.json`
- `public/uploads`

This works for local development, but Vercel serverless deployments do not provide persistent project-disk storage at runtime. For production admin editing and image uploads, connect one of these:

- Vercel Postgres / Supabase / Neon for CMS data
- Vercel Blob / S3 / Cloudflare R2 for uploaded images

The public website, RFQ form and email sending can deploy to Vercel normally. The local-file admin panel is suitable for local editing before deployment, or as a prototype before adding persistent storage.

## 2. Required Environment Variables

Add these variables in Vercel:

| Name | Required | Example | Purpose |
| --- | --- | --- | --- |
| `SMTP_HOST` | Yes | `smtp.gmail.com` | SMTP server host for RFQ emails |
| `SMTP_PORT` | Yes | `465` | SMTP port |
| `SMTP_SECURE` | Yes | `true` | Use SSL/TLS. Use `true` for port 465 |
| `SMTP_USER` | Yes | `your-email@example.com` | SMTP login account |
| `SMTP_PASS` | Yes | `your-app-password` | SMTP password or app password |
| `SMTP_FROM` | Yes | `Industrial Spring RFQ <your-email@example.com>` | Sender shown in RFQ emails |
| `RFQ_TO` | Yes | `fatmaachab862@gmail.com` | Receiver for RFQ inquiries |
| `ADMIN_PASSWORD` | Yes | `replace-with-strong-password` | Password for `/admin` |

## 3. Add Environment Variables in Vercel Dashboard

1. Open Vercel Dashboard.
2. Select your project.
3. Go to `Settings`.
4. Open `Environment Variables`.
5. Add each variable above.
6. Select environments: `Production`, `Preview`, and `Development` as needed.
7. Redeploy after saving variables.

## 4. Deploy from GitHub

1. Push this project folder to a GitHub repository.
2. Open Vercel Dashboard.
3. Click `Add New...` then `Project`.
4. Import the GitHub repository.
5. Vercel should detect `Next.js` automatically.
6. Keep these settings:
   - Framework Preset: `Next.js`
   - Install Command: `npm install`
   - Build Command: `npm run build`
   - Output Directory: leave default
7. Add the environment variables.
8. Click `Deploy`.

## 5. Deploy with Vercel CLI

Install Vercel CLI:

```bash
npm install -g vercel
```

Login:

```bash
vercel login
```

Deploy preview:

```bash
vercel
```

Deploy production:

```bash
vercel --prod
```

## 6. Local Test Before Deploy

```bash
npm install
npm run build
npm run dev
```

Open:

```bash
http://localhost:3000
```
